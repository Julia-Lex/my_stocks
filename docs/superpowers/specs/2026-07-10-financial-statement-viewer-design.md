# 财报查询页面设计(Financial Statement Viewer)

日期:2026-07-10
状态:已确认(用户批准)

## 目标

一个本机网页:输入股票名字(或代码),查询该股票的三大财务报表。支持 A股、港股、美股三个市场。

## 用户确认的需求

- **展示粒度**:摘要卡片 + 三大报表默认关键科目,可展开完整科目明细
- **报告期**:近 3 年年报 + 今年已披露各期(如 2026Q1),列为透视表格
- **市场范围**:A股 + 港股 + 美股(港美股财报覆盖有限,需要空态提示)

## 技术方案

单文件 FastAPI 后端 + 原生 JS 内嵌前端,无构建步骤:

- `webapp/app.py` — FastAPI 应用,psycopg 直连本机 `astock` 库(复用 `common.py` 的连接约定)
- `webapp/static/index.html` — 单页前端(原生 JS + CSS,无框架)
- 启动:`uv run uvicorn webapp.app:app --port 8500`

## 数据模型(现有,只读)

| 市场 | 股票基本表 | 财报表 | 备注 |
|---|---|---|---|
| A股 | `stock_basic(stock_code, name)` | `fin_statement` | 无 currency 列,人民币 |
| 港股 | `hk_stock_basic` | `hk_fin_statement(currency)` | 仅约 10 只有财报 |
| 美股 | `us_stock_basic` | `us_fin_statement(currency)` | 仅约 19 只有财报 |

财报表结构统一:`(stock_code, report_date, stmt_type['income'|'balance'|'cashflow'], ann_date, data jsonb)`。
jsonb 键均为中文科目名,但**三个市场科目名不同**(如 A股"归属于母公司所有者的净利润"、港股"归属母公司净利润"、美股"归属于母公司股东净利润"),关键科目需按市场分别配置候选键列表(取第一个存在的键)。

## API

### `GET /api/search?q=<关键词>`

- 联合查询三个市场 `*_stock_basic`,`name ILIKE %q%` 或 `stock_code ILIKE %q%`
- 返回:`[{code, name, market: 'cn'|'hk'|'us', has_statements: bool}]`,最多 10 条,A股优先
- `has_statements` 通过 EXISTS 子查询财报表得出,用于前端标灰无财报的港美股

### `GET /api/statements?market=<cn|hk|us>&code=<stock_code>`

- 报告期选择:近 3 个自然年的年报(12-31)+ 当前年已披露的所有报告期
- 返回:
  ```json
  {
    "code": "300308.SZ", "name": "中际旭创", "market": "cn", "currency": "CNY",
    "periods": ["2023-12-31", "2024-12-31", "2025-12-31", "2026-03-31"],
    "summary": {"营业总收入": ..., "归母净利润": ..., "基本每股收益": ...,
                 "经营现金流净额": ..., "总资产": ..., "资产负债率": ...},  // 最新期
    "statements": {
      "income":   {"key_items": [...科目名列表...], "rows": {"科目名": {"2023-12-31": 值, ...}}},
      "balance":  {...},
      "cashflow": {...}
    }
  }
  ```
- `rows` 含 jsonb 全部数值科目(过滤掉 币种/类型/数据源/公告日期/是否审计/更新日期 等元数据键);`key_items` 是该市场配置的关键科目中实际存在的,前端据此渲染默认视图
- 无任何财报数据 → `404 {"detail": "该股票暂无财报数据"}`

### `GET /`

返回 `static/index.html`。

## 前端

1. **搜索框**:输入防抖 300ms 调 `/api/search`,下拉候选显示 名称 + 代码 + 市场徽标(A股/港股/美股),无财报的候选置灰标注"无财报"
2. **摘要卡片区**:6 张卡片(营收、归母净利、EPS、经营现金流、总资产、资产负债率),显示最新报告期数值;金额按"亿"格式化,EPS 和负债率原值
3. **报表区**:三个 tab(利润表/资产负债表/现金流量表),每个 tab 一张透视表:行=科目、列=报告期(时间升序);默认只渲染 `key_items` 行,"展开全部科目"按钮切换为全部行(科目按 jsonb 原序,数值科目)
4. **细节**:港美股在表头显示币种;负数标红;空值显示"—"
5. **错误处理**:搜索无结果 → 下拉显示"无匹配";选中无财报股票 → 内容区提示"该股票暂无财报数据(数据库目前仅覆盖部分港/美股财报)";API 异常 → 页面顶部红色错误条

## 关键科目配置(app.py 内常量)

每个市场 × 每张表一个有序列表,元素为候选键元组(别名 fallback)。示例(A股):

- income: 营业总收入、营业成本、研发费用、营业利润、利润总额、净利润、归属于母公司所有者的净利润、基本每股收益
- balance: 资产总计、负债合计、货币资金、存货、应收账款、固定资产、所有者权益合计
- cashflow: 经营/投资/筹资活动产生的现金流量净额、期末现金及现金等价物余额

港股/美股按实际 jsonb 键另行配置(实现时从库中抽样确认)。
