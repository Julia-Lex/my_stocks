# 港股/美股日线数据库设计(方案 B:分市场独立表)

日期:2026-07-09
状态:已与用户确认方案 B(分表);本文档待用户评审

## 背景与目标

在现有 A 股日线库(PostgreSQL `astock`,AKShare/东财数据源)基础上,新增**港股**和**美股**的历史日线与后续增量。

用户决策记录:

- **架构:方案 B(每市场独立表)**。理由:不同市场适配不同策略,跨市场 UNION 非高频动作;隔离后货币/成交量单位/交易日历天然分开。
- **港股范围:全部 ~2,700 只**(主板+创业板),与 A 股同样保留全市场截面、防幸存者偏差。
- **美股范围:主要成分股 ~650-700 只** = 东财美股列表按总市值前 600 + 东财"知名中概股"板块,去重。

## 不变的设计原则(继承 A 股)

- 只存**不复权**原始价,后复权因子单独存;前/后复权通过视图动态算,绝不落地前复权价。
- 日线按 `trade_date` 年度 RANGE 分区;主键 `(stock_code, trade_date)` + 反向索引 `(trade_date, stock_code)`。
- 价格 `NUMERIC` 不用 `FLOAT`;退市/掉出清单的股票保留历史不删。
- 周线/月线用物化视图从后复权日线聚合,单一事实来源。
- `etl_progress` 断点续传(复用现有表,task 名区分:`init_hk` / `init_us`)。

## Schema(新文件 `04_schema_hk_us.sql`)

每市场一套镜像结构,前缀 `hk_` / `us_`:

| 表/视图 | 说明 |
| --- | --- |
| `hk_stock_basic` / `us_stock_basic` | 基本信息,含 `delist_date`。美股多一列 `em_symbol`(东财拉数代码,如 `105.AAPL`)。 |
| `hk_daily_price` / `us_daily_price` | 年度分区(港股 1980~2030,美股 1970~2030;插入越界年份会报错,提前留足)。结构同 A 股 `daily_price`。 |
| `hk_adj_factor` / `us_adj_factor` | 后复权因子 = 东财 hfq 收盘价 ÷ 原始收盘价。 |
| `hk_trade_calendar` / `us_trade_calendar` | **从指数日线派生**(指数有数据的日子即交易日),无独立日历源。 |
| `hk_index_daily` / `us_index_daily` | 港:恒生指数、恒生科技;美:标普500、纳斯达克综合、道琼斯。 |
| `hk_daily_price_hfq/qfq` 等视图 + `hk_weekly/monthly_price_hfq` 物化视图 | 定义照抄 A 股,改前缀。美股同。 |

代码规范:`00700.HK`、`AAPL.US`(即使分表也带后缀,防止未来合并分析时歧义)。

单位与货币(记入 README):

- 成交量:港/美股单位为**股**(A 股为手,1 手=100 股)。
- 货币:按表隐含——`hk_*` 为 HKD,`us_*` 为 USD(中概股在美上市也是 USD),不设 currency 列。

## 数据范围与清单策略

- 港股:`stock_hk_spot_em()` 全列表(~2,700 只)。
- 美股:`stock_us_spot_em()` 按总市值降序取前 600 + 知名中概股板块,去重。
  - 清单为**快照式**:成分变化时,`us_stock_basic` 只增不删;掉出清单的停止增量但保留历史(防幸存者偏差)。
  - 实现时若"知名中概股"接口名与预期不符(AKShare 接口易变),以东财中概板块等价接口替代,兜底方案:市值前 600 已覆盖主要中概(BABA/PDD/NTES 等)。

## ETL

- `common.py`:新增 `MARKETS` 配置字典(表前缀、拉取函数、代码映射、指数列表);A 股既有代码路径不动。
- `05_init_load_intl.py --market hk|us [--workers N] [--limit N] [--reset]`:全量初始化。复用 02 已实现的线程池并行(每线程独立连接)与 `etl_progress` 断点续传。
- `06_daily_update_intl.py --market hk|us`:每日增量 + 自动补漏,逻辑同 `03_daily_update.py`。
- 接口调用沿用指数退避重试(2s→4s→8s→16s);已实测限流特征:5 并发内失败率 ~10-12%,失败股票重跑自动补。

## 更新节奏(北京时间)

| 市场 | 收盘 | cron |
| --- | --- | --- |
| A 股 | 15:00 | 工作日 18:00(已有) |
| 港股 | 16:10 | 工作日 18:00,与 A 股同一趟 |
| 美股 | 次日 04:00(夏令)/05:00(冬令) | 工作日 09:00 拉前一交易日 |

注意:盘中拉取会得到未定盘的当日 K 线(实时价/不完整成交量);upsert 主键覆盖保证收盘后重跑即修正。

## 错误处理

- 单只失败:记 `etl_progress status='error'`,不中断整体;重跑自动重试 error 与未完成的。
- 接口列名漂移:集中在 `common.py` 的 RENAME 映射,与 A 股同一处理方式。
- 指数拉取失败导致日历缺失:日历派生自指数,若当日指数未更新,增量脚本按"最后已知交易日+1"补拉,幂等无副作用。

## 不做的事(YAGNI)

- 不做分钟线、财务数据(与 A 股二期一并考虑)。
- 不做汇率表/统一货币折算(策略按市场隔离,暂无跨市场估值需求)。
- 不做标普500/纳指100 官方成分历史(无稳定免费源;市值快照近似即可)。
