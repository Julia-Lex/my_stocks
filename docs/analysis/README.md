## MiniMax(00100.HK)解禁抛压分析(2026-07-11)

- 报告:`2026-07-11-minimax-unlock.html`(数据截至 2026-07-10,即解禁后仅 2 个交易日)
- 脚本:`unlock_analysis.py`(事件研究)→ 产出 `unlock_data.json` → `unlock_project.py`(抛压外推,把 projection 追加进同一 JSON)
- 运行:`ASTOCK_DB_USER=zhu .venv/bin/python docs/analysis/unlock_analysis.py`(只读查询,任何时间可跑)

### 方法口径(续作时保持一致,否则数字不可比)

- **事件定义**:港股基石/上市前投资者解禁日 = 上市日 + 6 个月,顺延至下一交易日(未逐一核对公告,对照组按此推算;MiniMax 实际解禁日 2026-07-09 已被新闻证实)。
- **基线成交量** = 解禁日前第 25~6 个交易日的中位数(避开解禁前一周抢跑放量)。MiniMax 基线 ≈ **220 万股/日**。
- **超额成交量** = max(当日量 − 基线, 0),作为解禁抛售的**代理上限**(含跟风换手、不含大宗/场外)。
- **量能消化完毕** 判定:5 日均量 < 1.2× 基线。
- **外推模型**:vol_mult(k) = 1 + (m1−1)·exp(−λ·(k−1)),快衰减 λ=0.521(MiniMax 自身 T0→T1),慢衰减 λ=0.089(对照组高倍数子组:宁德/三花/蓝思拟合,半衰期 7.8 天)。

### 关键常数(来自公开报道,新浪科技/证券时报 2026-07-09)

- 解禁约 **1.53 亿股 = 总股本 48.9%**(总股本 ≈ 3.13 亿股);财务投资者约占解禁量 63%(中金口径),阿里/米哈游/国寿等 80%+ 股东表态继续持有;创始团队另锁 12 个月(下一批解禁 ≈ 2027-01)。
- 上市 2026-01-09,发行价 ≈165 港元;收盘峰值 1238(2026-03-16)。

### 截至 2026-07-10 的结论快照

- 已观测超额成交 ≈ 2,990 万股 ≈ 解禁量 **19.5%**;外推最终消化 30%(T+10)~97%(T+40),结合股东表态最可能 30%~60%,即 **7 月下旬~8 月中旬消化完毕**。
- 对照组规律:T0 量能倍数越高消化越慢(高倍数组回归中位 T+26);60 日中位最大回撤 ≈ −17%,低点多在解禁后第 2 个月;高倍数组 60 日相对恒指显著跑输。
- **跟踪指标:每日成交量 ÷ 220 万股基线**,回落到 1.2× 以内且价格止跌 = 本轮消化完毕;若 8 月中旬仍 >2× 按悲观情景处理。

### 续作方式

1. 重跑两个脚本得到新 `unlock_data.json`(MiniMax/智谱的 post 窗口会自动变长)。
2. 报告 HTML 中数据是内嵌的:把 `const DATA = {...};` 整段替换为新 JSON 即可(报告最初由模板占位符 `__DATA__` 注入生成,模板未保留)。
3. 值得复核的点:量能衰减率 λ 用更多交易日重估;对照组解禁日如需精确可查各家公告;智谱(02513.HK)满 60 日后可并入对照组统计。

### 已知局限

- 港股日线(腾讯源)无成交额/换手率字段,全部用股数口径。
- MiniMax 解禁占股本比例(48.9%)数倍于对照组基石解禁,对照映射偏乐观方向。
- `unlock_data.json` 是中间产物,不入库(已在 .gitignore 逻辑之外,勿提交)。
