## 2. 数据源特征与限流

### 东财(EastMoney)
- 基线:单/低并发也有 ~10% RemoteDisconnected,指数退避可自愈。
- 硬封禁:**5 并发 × 约 2 小时**(数万请求)触发连接级封禁,所有请求(含浏览器头直连)一律被断;新浪等其他源不受影响。
- 封禁范围是"行情族"(push2 + push2his):历史 K 线、**实时列表 spot 接口**(2026-07-11 实测 `stock_hk_spot_em` 同样被拒)都在内;**datacenter 接口不受影响**(基本面/估值初始化 3 并发 25 分钟实测安全)。
- 二次封禁持续过 13+ 小时;**封禁期间任何持续重试都可能续期封禁**;解封判定要"连续 2 次探测通过"(滚动解封有单次假阳性),探测降频到 1 小时/次。
- 安全节奏:全量拉取 ≤3 并发、分市场错峰、长任务中途歇息;被封后停止硬试,靠断点续传零浪费。
- A/港/美个股行情同为 push2his(共享同一 IP 预算);指数走新浪,不受东财封禁影响。

### 腾讯(qt.gtimg.cn / web.ifzq.gtimg.cn)
- K 线备源(`ASTOCK_ASHARE_SOURCE=tx` / 港美主源),有 WAF:请求节流 ≥0.35s(common.py `_tx_throttle`)。
- **A股成交量单位陷阱:科创板(688/689)返回股,主板/创业板返回手**(已在 `_fetch_daily_tx` 按板块换算);北交所无 K 线。
- fq 参数对港/美股不生效,港美复权因子走新浪(见 common.py 注释)。
- 实时行情接口(qt.gtimg.cn/q=)用于 webapp 港美股估值:港股 f39=PE、f58=PB、f47=股息率、f44=总市值(亿);美股 f39=PE、f47=EPS、f44=市值。

### 新浪
- 指数日线、复权因子(A股/港股 hfq-factor、美股 qfq-factor 需重锚定)、港股全量列表(`stock_hk_spot`,2798 只含中文名)。

### 富途(用户有账号)
- OpenAPI 需本地常驻 FutuOpenD 网关。`get_financials_statements` 历史深度惊人(港股 2001/美股 1979/A股 1998),已是港美基本面主源;无公告日(ann_date 靠东财)。
- 限频:通用 30 次/30 秒,但**各接口可能有独立限频**(`get_plate_stock`、`request_history_kline` 实测均 10 次/30 秒,官方通用页未标注);新接口首用留意 ret=-1 报错文案里的真实配额。common.py 里两接口各配独立 3.1s 节流时钟(`_futu_pace`)。
- 历史 K 线"每 30 天可下载股票数"额度(本账号 1000 标的/月):**A 股板块层已用 ~918**(2026-07-11 全量),**跨会话分配约定:港美板块链 0 额度(只用列表+成分),K 线额度归 A 股板块**;下月额度重置时 A 股板块日增量会再计 ~918。
- **板块接口坑位**(2026-07-11 实测):板块代码 `request_history_kline` 的 `end=None` 返回 0 行(个股正常),必须显式传日期;富途板块列表含少量"僵尸残留"(列表有、行情系统不认,无指数无成分,报"未知股票"),按 is_active=false 处理;`OpenQuoteContext` 有非守护线程,脚本 finally 必须 `close_futu()`。
- **跨会话限频窗口约定(2026-07-11)**:工作日 **18:50–19:20 归港美板块链**(get_plate_stock 密集 ~27 分钟);**A 股板块 21_board_update.py cron 定 19:30**,互不进对方窗口;临时手跑富途密集任务也要避开对方窗口。

## 3. 全库数据约定

- **成交量单位全库统一为"股"**(2026-07-09 定案):东财/腾讯 A股原始是手,在 `common.py` 入库层 ×100;港/美与指数原生股。教训:改单位必须数据/代码/文档三层同步。
- **复权因子锚点三市场统一"最早日=1、逐笔递增"**;美股新浪 qfq-factor 锚点在最新日,入库前重锚定(common.py)。港股新浪因子忽略现金分红列(已知简化:只保证拆合股连续,分红不调整——与富途/同花顺前复权存在系统性差异,历史越久越明显)。
- **hfq 视图精度 6 位小数**(2026-07-11 修复):大比例合股仙股(如 00661.HK 因子 0.0075)后复权价低至 1e-4,3 位小数会舍成 0 并固化进周/月物化视图。
- 周/月线:`*_weekly/monthly_price_hfq` 物化视图(后复权),日更脚本 `REFRESH CONCURRENTLY`;前复权 = 视图值 ÷ 该股最新因子。
- 港股中文名:`hk_stock_basic.name_cn`(HKEX 官方 name 是英文缩写如"VGT"=胜宏科技);源优先级 东财 spot → 新浪列表 → 腾讯 A+H 对照(common.fetch_hk_names_cn),06 日更幂等刷新。
- 收盘防护(A股 15:30 口径):`safe_cutoff_date()` 取网络/本机双时钟较保守值、10 分钟 TTL、全程留痕;`drop_unclosed_bars()` 挂在 upsert 层。
- **A 股板块层双源**(基本面二期分支,用法详见 README"板块数据"节):`ASTOCK_BOARD_SOURCE=em|futu`,`board.source` 列区分同库共存。富途口径 2026-07-11 全量已入库(行业 131+概念 792,日线 2018 起 122 万行,成分区间 10.5 万);东财口径(官方五档资金流、日线 ~2006 起)待行情族解封后由守候脚本自动补;板块资金流已有派生口径:物化视图 board_capital_flow = 个股 capital_flow × 当前成分求和(近 1 年,当前成分回算近似)。成分区间表 `valid_from`=观测日(建库日 2026-07-11),as-of 查询早于观测起点返回空是正确行为。

## 4.2 待办-报告工作流(2026-07-12 起)

用户在 webapp「待办事项」页记录分析需求(`todo` 表);分析会话完成一条后:

1. 报告写成**自包含 HTML**(内联样式/脚本,无外部依赖),放 `docs/analysis/YYYY-MM-DD-<slug>.html`,合入 main;
2. 挂接到待办并标完成(webapp 常驻 8500 端口):
   `curl -X PATCH localhost:8500/api/todos/{id} -H 'Content-Type: application/json' -d '{"done": true, "report": "YYYY-MM-DD-<slug>.html"}'`
   (待办 id 用 `GET /api/todos` 查;report 只填文件名,页面经 `/reports/{name}` 提供)
3. 页面上该待办即显示「📄 查看分析报告」链接。

注意:webapp 运行目录的 `docs/analysis/` 必须包含该文件——报告合入 main 后,提醒可视化会话把 main 合进 webapp 分支(两者同分支则无需)。
